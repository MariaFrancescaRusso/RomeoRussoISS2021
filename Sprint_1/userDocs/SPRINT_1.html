Planner

The planning feature is the capability to execute actions in a logical space; this space is a map described by a set of square cells whose edges are robot sized. So it's possible to see a step of the robot as the ingress in a new cell. The planner keeps track of the current position in the map .
The planner feature is documented in the LabPlanner LabPlanner2021.
Support is provided by the it.unibo.planner20-1.0.jar library and it's available in itunibo.planner.plannerUtil. The main operations provided by the planner are:
  • initAI(): initializes the planner assuming that the robot is in the robot home position.
  • setGoal(x:int, y:int): allows the system to set a goal to be reached providing the coordinates of the goal cell.
  • doPlan(): List?: allows the system to get the moves to reach the goal from the current position.
  • doMove(move:String): allows the system to perform a move in the logical space.
    • There are other functions that allow the system to manage the map used by the planner. In this sprint are very useful:
  • loadRoomMap: allows the system to load a map.
  • get_curPos : Pair: return the current position of the robot in the map.
